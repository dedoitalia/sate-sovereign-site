Developer integration

I tuoi developer parlano già la sua lingua.

SATE Sovereign si integra in modo nativo con OpenHands (l'agente autonomo open-source) e con i modelli Hermes di Nous Research. Il tuo team continua a usare gli strumenti che conosce; cambia solo l'endpoint, e tutto resta in casa tua.

OpenHands · Autonomous Agent

Un agente che lavora al tuo fianco.

OpenHands (ex OpenDevin) è l'agente AI open-source più adottato per software engineering autonomo. Esegue task multi-step nel tuo sandbox: legge codice, scrive PR, esegue test, naviga la documentazione. SATE Sovereign è il backend perfetto: tutto sotto la tua VPN, niente CLOUD Act.

  • Drop-in con base_url SATE Sovereign — zero cambi di codice
  • Sandbox Docker in azienda, niente prompt verso USA
  • PR autonome su GitLab/Gitea/Bitbucket on-prem
  • Ticket → branch → PR → review umana finale
# config.toml — il tuo OpenHands punta a SATE Sovereign [llm] model = "sovereign-pro-120b" base_url = "https://api.sate-sovereign.eu/v1" api_key = "sk-sov-..." temperature = 0.2 [sandbox] workspace_mount_path = "/srv/plc-projects" enable_browser = true
Hermes · by Nous Research

Modelli pensati per gli agenti.

La famiglia Hermes (3 e 4) di Nous Research è una delle migliori per tool calling, function calling e workflow agentici. Open-weight, steerable, addestrata per istruzioni complesse e output strutturati. Su SATE Sovereign puoi servirla privatamente con la stessa API OpenAI compatibile.

  • Tool calling robusto su JSON Schema
  • Output strutturato deterministico (XML, JSON, regex)
  • Steerable: niente refusal su task tecnici legittimi
  • Open-weight: nessun lock-in di vendor
# Esempio: Hermes 4 405B con tool calling per analisi PLC curl https://api.sate-sovereign.eu/v1/chat/completions \ -H "Authorization: Bearer $SOV_KEY" \ -d '{ "model": "sovereign-hermes-4-405b", "messages": [{"role":"user","content":"Analizza il log..."}], "tools": [{"type":"function","function":{...}}], "tool_choice": "auto" }'
